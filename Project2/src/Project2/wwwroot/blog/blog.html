Learning Vim

January 16, 2016 by Afton

When I first started working in Vim I was intimidated. However with some tools I'm going to talk about I hopefully can help you get up to speed fairly quickly.

vimtutor

Hopefully already installed on your distro, this handy tutorial is interactive and encourages you to edit from within the tutorial. You can access this by typing in 'vimtutor' (no quotes) and pressing enter. Pretty cool, and my recommended first step for beginners

The Linux Command Line

Besides an excellent resource for learning the basics of the linux command line, it also includes a gentle introduction to vi/vim. William Shotts is great at presenting a broad-strokes tour of linux, and if you like his free book, feel free to buy the paper version.

My Favorite Textbooks from College

January 15, 2016 by Afton

Kind of weird, but I'd thought I'd list my favorite books and/or textbooks from attending UCO.

Programming: Principles and Practices using C++

For anyone that asserts C++ is not for beginners I present to you the creator of C++ and is book explicitly for beginners. If you think this is the book that's normally mentioned by him, The C++ Programming Language, you can be forgiven. But also realize this book is SO much easier, and the questions in the book really encourage you to think about different scenarios and start to think logically like a programmer.

The C Programming Language

This book is an elegant intro to C. I wouldn't really recommend it to an absolute beginner, but if you watched some youtube tutorials before digging into this surprisingly concise book, it would really benefit you. It's constantly referenced as a great example of technical writing and is used as the standard that technical tutorial style books try to emulate.

Computer Systems: A Programmer's Perspective

Really good book, and in my opinion would have served me better than my hardware based classes, as its focused on techniques that are important and useful to be a great programmer, rather than a systems engineer. Optimization, compiler techniques and a dab of assembly: it really hits the sweet spot between computer architecture and software engineering.

Java Design Pattern Essentials

After trying my brain with the GOF book on design patterns, I really wanted and hoped that there was a more dumbed down resource for explaining design patterns that was concise and read less like a white-paper. That and the fact that I was using Java, which I've heard was designed the intention of strictly conforming to GOF's ideas about design patterns. This book is so small, yet so effective in getting the basic idea of the pattern across. It does this with effective code naming and family, species, or real-life-type relationships rather than long explanations(GOF) or entertaining anecdotes (Head First Design Patterns).
